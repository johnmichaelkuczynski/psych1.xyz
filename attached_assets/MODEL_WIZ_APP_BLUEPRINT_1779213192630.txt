═══════════════════════════════════════════════════════════════════════════════ ARCHITECTURE NOTES ═══════════════════════════════════════════════════════════════════════════════

LLM PROVIDER MAPPING (ZHI 1-5) ZHI 1 = OpenAI (GPT-4o / gpt-4o-mini) env: OPENAI_API_KEY ZHI 2 = Anthropic (claude-sonnet-4-5) env: ANTHROPIC_API_KEY ZHI 3 = DeepSeek (deepseek-chat) env: DEEPSEEK_API_KEY ZHI 4 = Perplexity (sonar) env: PERPLEXITY_API_KEY ZHI 5 = Grok / xAI (grok-3) env: GROK_API_KEY

Auto-fallback chain (in server/routes.ts inline logic and llmRouter.ts): ZHI 5 → ZHI 2 → ZHI 1 → ZHI 3 → ZHI 4 Used when a provider returns 429, 5xx, or times out.

UI label rule (user preference): ZHI labels MUST NOT include parenthetical provider names (e.g. show "ZHI 2", never "ZHI 2 (Anthropic)").

THE FULL SUITE PIPELINE (handleRunFullSuite in HomePage.tsx ~line 905) Stage 1: Batch Analysis (5 modes, parallel with concurrency=2) reconstruction, isomorphism, mathmodel, truth-isomorphism, math-truth-select (+ axiomatic-transform if user selected it) Stage 2: BOTTOMLINE synthesis from all batch outputs Stage 3: Objections — 25 likely objections + counter-arguments Stage 4: Refined Rewrite — BOTTOMLINE rewritten in light of objections Output: Single .txt download via buildFullSuiteDocument() ("Download Everything as One Document" button)

DIAGNOSTIC (auto-runs when user clicks "Diagnostic" tab) Backend: POST /api/diagnostic/run (server/routes.ts ~line 597) Frontend: client/src/pages/DiagnosticPage.tsx (useEffect auto-start) Checks (18 total): Environment (9): OPENAI/ANTHROPIC/DEEPSEEK/PERPLEXITY/GROK keys, DATABASE_URL, plus optional MATHPIX/SENDGRID/STRIPE Database (1): PostgreSQL connectivity via storage.getUser(1) AI Providers (5): tiny ping to each configured provider (HTTP 429 reported as WARN, not FAIL) Pipeline (3): batch / bottomline / objections with tiny test input

PAYMENTS (optional) STRIPE_SECRET_KEY missing is OK — payment routes are null-safe. Per-provider credit pricing is configured in server/lib/stripe-config.ts.

USER PREFERENCES (also in replit.md) • Plain, everyday language in UI copy • NEVER show raw error messages (e.g. "Error: 404 {...}") — always friendly • ZHI labels: no parenthetical provider names • Default audience/objective to "Educated audience looking for actionable business ideas" if user leaves them blank • Don't block user with validation toasts • JMKUCZYNSKI username = admin bypass (no password required)

FORBIDDEN / LOCKED FILES • package.json — do not edit; install via package manager only • vite.config.ts — do not modify • server/vite.ts — do not modify • drizzle.config.ts — do not modify
